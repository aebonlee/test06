## 개요

### 목적
HDH Fintech 플랫폼을 원활하게 운영하기 위해 관리자가 수행해야 하는 모든 수동 작업을 정리

### 전제 조건
- Admin Dashboard 접근 권한 보유
- 데이터베이스 관리자 권한
- 이메일 발송 권한
- 교육 담당자 연락처

### 작업 우선순위
1. 🔴 **긴급 (Urgent):** 즉시 처리 (30분 이내)
2. 🟡 **중요 (Important):** 당일 처리
3. 🟢 **일반 (Normal):** 2-3일 이내 처리

---

## 일일 업무 (Daily Operations)

### 1. 아침 루틴 (09:00 - 09:30)

#### 1-1. Admin Dashboard 개요 확인

**접속:** Admin Dashboard → 📊 대시보드 개요

**확인 항목:**
- ✅ 총 회원 수 (전일 대비 증감)
- ✅ 미처리 문의 개수 (Sunny 챗봇 + 고객센터)
- ✅ 교육 신청 대기 건수
- ✅ 추천인 코드 신청 대기 건수

**예시 화면:**
```
┌─────────────────────────────────────┐
│ 총 회원 수: 456명                   │
│ ↑ +3명 (어제 대비)                  │
├─────────────────────────────────────┤
│ 미처리 문의: 5건                    │
│ ⚠️ 긴급 1건                         │
├─────────────────────────────────────┤
│ 교육 신청 대기: 2건                 │
│ 추천인 코드 신청 대기: 4건          │
└─────────────────────────────────────┘
```

**액션:**
- 긴급 문의 1건 확인 → 즉시 처리
- 교육 신청 2건 → 오전 중 연락
- 추천인 코드 신청 4건 → 교육 이수 여부 확인 후 승인

---

#### 1-2. 최근 활동 타임라인 검토

**위치:** 대시보드 개요 → 📋 최근 활동

**확인 항목:**
```
🟢 신규 가입: 김철수 (15분 전)
🟡 교육 신청: 이영희 (1시간 전)
🔴 긴급 문의: API 연결 오류 (2시간 전)
🟢 추천인 코드 신청: 박민수 (3시간 전)
```

**액션:**
- 🔴 긴급 항목 → 즉시 처리
- 🟡 중요 항목 → 메모 후 오전 중 처리
- 🟢 일반 항목 → 확인만

---

#### 1-3. 긴급 처리 필요 항목 확인

**위치:** 대시보드 개요 → ⚠️ 긴급 처리 필요

**우선순위 목록:**

| 우선순위 | 항목 | 건수 | 예상 소요 |
|---------|------|------|----------|
| 🔴 High | API 연결 오류 문의 | 3건 | 30분 |
| 🟡 Medium | 교육 신청 (신규) | 5건 | 1시간 |
| 🟡 Medium | 추천인 코드 신청 | 8건 | 40분 |
| 🟢 Low | 일반 문의 | 10건 | 1시간 |

**액션:**
- 각 항목 클릭 → 해당 섹션으로 이동
- 우선순위대로 처리 계획 수립

---

### 2. 교육 신청 처리 (09:30 - 10:30)

#### 2-1. 교육 신청 목록 확인

**접속:** Admin Dashboard → 📚 교육 관리 → 신청 목록

**필터 적용:** 상태 = "신청 접수"

**테이블 예시:**
```
┌──────────┬─────────┬──────────────┬──────────┬────────┐
│ 신청 ID  │ 이름    │ 이메일       │ 전화번호 │ 상태   │
├──────────┼─────────┼──────────────┼──────────┼────────┤
│ EDU-001  │ 김철수  │ kim@xxx.com  │ 010-1234 │ 접수   │
│ EDU-002  │ 이영희  │ lee@xxx.com  │ 010-5678 │ 접수   │
└──────────┴─────────┴──────────────┴──────────┴────────┘
```

---

#### 2-2. 신청자에게 연락

**Step 1: 신청 상세 확인**
- 신청 ID 클릭
- 수강 동기, 트레이딩 경험, 기대하는 점 확인

**Step 2: 전화 또는 이메일 연락**

**전화 스크립트:**
```
"안녕하세요, HDH Fintech입니다.
아카데미 교육 신청 주셔서 감사합니다.

교육 일정을 조율하고자 연락드렸습니다.
언제 시간이 괜찮으신가요?

[일정 협의]

교육 방식은 온라인/오프라인 중 선호하시는 방식이 있으신가요?

[확인 후]

그럼 [확정 날짜]에 [방식]으로 진행하겠습니다.
교육 자료는 이메일로 미리 보내드리겠습니다.
감사합니다!"
```

**이메일 템플릿:**
```
제목: [HDH Fintech] 아카데미 교육 일정 안내

안녕하세요, HDH Fintech입니다.

아카데미 교육 신청 감사드립니다.

교육 일정을 안내드립니다:
- 날짜: 2025년 12월 15일 (토)
- 시간: 오전 10시 - 오후 5시
- 방식: 온라인 (Zoom 링크 별도 발송)
- 준비물: 노트북, 필기도구

교육 자료는 교육 시작 1일 전에 보내드리겠습니다.

문의사항이 있으시면 언제든 연락주세요.

감사합니다.
HDH Fintech 팀
```

---

#### 2-3. Admin Dashboard에서 상태 업데이트

**접속:** Admin Dashboard → 교육 관리 → 신청 상세

**업데이트:**
- 상태: "신청 접수" → "일정 확정"
- 교육 날짜 입력
- 교육 방식 선택 (온라인/오프라인)
- 메모 추가 (필요 시)

**[저장]** 클릭

---

### 3. 교육 진행 및 이수 처리 (교육 당일 + 다음날)

#### 3-1. 교육 진행

**교육 당일:**
- 교육 자료 준비
- Zoom 링크 발송 (온라인인 경우)
- 교육 진행

**교육 시간:** 6-7시간 (휴식 포함)

---

#### 3-2. 교육 이수 처리

**교육 완료 후:**

**Step 1: Admin Dashboard에서 이수 처리**
- 접속: Admin Dashboard → 교육 관리 → 교육생 목록
- 해당 교육생 찾기
- 상태: "교육 진행 중" → "교육 이수 완료"
- 이수 날짜 입력
- [저장]

**Step 2: 시스템 자동 트리거**
- ✅ 교육 이수 완료 이메일 자동 발송
- ✅ 추천인 코드 신청 자격 부여 (시스템 내부 플래그)
- ✅ 사용자 Dashboard에 "추천인 코드 신청 가능" 표시

**이수 완료 이메일 (자동 발송):**
```
제목: [HDH Fintech] 아카데미 교육 이수 완료 축하드립니다!

안녕하세요, HDH Fintech입니다.

아카데미 교육을 성공적으로 이수하셨습니다!
축하드립니다! 🎉

이제 다음 단계로 진행하실 수 있습니다:
1. 추천인 코드 신청하기
2. 매뉴얼 숙지하기
3. AI 프로그램 다운로드
4. My Settings 설정

Dashboard에서 바로 시작하실 수 있습니다.

[Dashboard 바로가기]

감사합니다.
HDH Fintech 팀
```

### 4. 추천인 코드 신청 처리 (수시)

#### 4-1. 추천인 코드 신청 목록 확인

**접속:** Admin Dashboard → 👥 회원 관리 → 추천인 코드 신청

**필터 적용:** 상태 = "신청 대기"

**테이블 예시:**
```
┌──────────┬─────────┬──────────────┬────────────┬────────┐
│ 신청 ID  │ 이름    │ 이메일       │ 교육 이수  │ 상태   │
├──────────┼─────────┼──────────────┼────────────┼────────┤
│ REF-001  │ 김철수  │ kim@xxx.com  │ ✅ 이수    │ 대기   │
│ REF-002  │ 이영희  │ lee@xxx.com  │ ❌ 미이수  │ 대기   │
└──────────┴─────────┴──────────────┴────────────┴────────┘
```

---

#### 4-2. 교육 이수 여부 확인 및 승인/거부

**Case 1: 교육 이수 완료 (✅)**

**액션:**
- 신청 ID 클릭
- 신청 사유 확인
- [승인] 버튼 클릭

**승인 처리:**
1. 시스템이 자동으로 추천인 코드 생성 (예: HDH2025ABC123)
2. 상태: "신청 대기" → "승인 완료"
3. 자동 이메일 발송

**승인 완료 이메일 (자동 발송):**
```
제목: [HDH Fintech] 추천인 코드 발급 완료

안녕하세요, HDH Fintech입니다.

추천인 코드가 발급되었습니다!

🎯 나의 추천인 코드: HDH2025ABC123

📌 추천인 코드 용도:
- 회원 유입 경로 추적용
- 향후 통계 분석 자료로 활용

💡 참고:
추천인 코드는 순수 추적 목적이며,
별도 리워드는 없습니다.

[Dashboard 바로가기]

감사합니다.
HDH Fintech 팀
```

---

**Case 2: 교육 미이수 (❌)**

**액션:**
- 신청 ID 클릭
- 교육 이수 여부 확인 (❌ 미이수)
- [거부] 버튼 클릭
- 거부 사유 입력: "교육 미이수"

**거부 처리:**
1. 상태: "신청 대기" → "거부"
2. 거부 사유 이메일 발송

**거부 이메일 (자동 발송):**
```
제목: [HDH Fintech] 추천인 코드 신청 안내

안녕하세요, HDH Fintech입니다.

추천인 코드는 아카데미 교육 이수 후
신청 가능합니다.

현재 교육 이수 기록이 확인되지 않아
신청이 보류되었습니다.

아카데미 교육을 먼저 수강해주시기 바랍니다.

[교육 신청하기]

감사합니다.
HDH Fintech 팀
```

### 5. Sunny 챗봇 문의 모니터링 (수시)

#### 5-1. Sunny 챗봇 로그 확인

**접속:** Admin Dashboard → 💬 문의 관리 → Sunny 챗봇 로그

**확인 항목:**
- AI가 제대로 답변하지 못한 질문
- 사용자 피드백 (👍/👎)
- 자주 묻는 질문 패턴

**예시:**
```
┌─────────────┬───────────────────────┬──────────┬────────┐
│ 시간        │ 사용자 질문           │ AI 답변  │ 피드백 │
├─────────────┼───────────────────────┼──────────┼────────┤
│ 10:23       │ API 연결이 안 돼요    │ 답변 완료│ 👍     │
│ 11:15       │ 계좌 개설 어떻게?     │ 답변 완료│ 👍     │
│ 14:32       │ [복잡한 질문]         │ 답변 실패│ 👎     │
└─────────────┴───────────────────────┴──────────┴────────┘
```

---

#### 5-2. AI가 답변하지 못한 질문 처리

**문제 있는 질문 발견 시:**

**Step 1: 질문 상세 확인**
- 질문 클릭
- 전체 대화 내역 확인
- AI가 답변하지 못한 이유 파악

**Step 2: 직접 답변 또는 고객센터 안내**

**옵션 1: 직접 답변**
- [직접 답변하기] 버튼 클릭
- 답변 작성
- [전송] → 사용자에게 이메일/시스템 알림 발송

**옵션 2: 고객센터 문의 전환**
- [고객센터로 전환] 버튼 클릭
- 고객센터 문의 목록에 자동 추가
- 사용자에게 "고객센터에서 곧 답변 드리겠습니다" 알림

### 6. 고객센터 문의 처리 (10:00 - 12:00, 14:00 - 16:00)

#### 6-1. 고객센터 문의 목록 확인

**접속:** Admin Dashboard → 💬 문의 관리 → 고객센터 문의

**필터 적용:** 상태 = "미답변"

**테이블 예시:**
```
┌──────────┬─────────┬──────────┬──────────────────┬────────┐
│ 문의 ID  │ 이름    │ 유형     │ 제목             │ 상태   │
├──────────┼─────────┼──────────┼──────────────────┼────────┤
│ INQ-001  │ 김철수  │ 기술지원 │ API 연결 오류    │ 미답변 │
│ INQ-002  │ 이영희  │ 일반문의 │ 교육 일정 문의   │ 미답변 │
└──────────┴─────────┴──────────┴──────────────────┴────────┘
```

---

#### 6-2. 문의 상세 확인 및 답변

**Step 1: 문의 상세 보기**
- 문의 ID 클릭
- 문의 내용 전체 확인
- 첨부 파일 확인 (있는 경우)

**Step 2: 답변 작성**
- [답변하기] 버튼 클릭
- 답변 작성

**답변 예시 (API 연결 오류):**
```
안녕하세요, HDH Fintech입니다.

API 연결 오류에 대해 답변드립니다.

다음 순서로 확인해주세요:

1. My Settings에서 API 키 확인
   - API 키가 정확하게 입력되었는지 확인
   - 공백이나 특수문자 오입력 확인

2. 계좌 상태 확인
   - 프롭펌 계좌가 활성화되어 있는지 확인
   - 계좌 잔액 확인

3. 방화벽/보안 프로그램 확인
   - 방화벽이 연결을 차단하고 있지 않은지 확인

4. AI 프로그램 재시작
   - 프로그램을 완전히 종료 후 재실행

위 방법으로도 해결되지 않으면
원격 지원을 제공해드리겠습니다.
전화번호를 알려주시면 연락드리겠습니다.

감사합니다.
HDH Fintech 팀
```

**Step 3: 답변 전송**
- [답변 전송] 버튼 클릭
- 상태: "미답변" → "답변 완료"
- 사용자에게 이메일 자동 발송

### 7. 공지사항 등록 (필요 시)

#### 7-1. 공지사항 작성

**접속:** Admin Dashboard → 📢 공지사항 관리 → 새 공지 작성

**입력 항목:**
```
제목: [입력]
예: "시스템 점검 안내 (12월 15일)"

공지 유형: [선택]
- 📢 중요 공지
- 🔔 시스템 업데이트
- ⚠️ 긴급 공지

내용: [작성]
예:
안녕하세요, HDH Fintech입니다.

시스템 점검 일정을 안내드립니다.

- 점검 일시: 2025년 12월 15일 (일) 02:00 - 05:00
- 점검 사유: 서버 업그레이드
- 영향: 일시적으로 서비스 이용 불가

점검 중 불편을 드려 죄송합니다.

감사합니다.

상단 고정: [☑️ 고정 / ☐ 일반]

[등록]
```

**등록 완료:**

## 긴급 대응 (Emergency Response)

### 1. API 연결 장애

#### 대응 프로세스

**Step 1: 장애 확인**
- 여러 회원이 동시에 API 연결 오류 보고
- 시스템 모니터링 도구에서 알림

**Step 2: 긴급 공지**
- 공지사항 등록 (⚠️ 긴급 공지)
- "API 연결 장애 확인 중 - 빠른 시일 내 복구 예정"

**Step 3: 원인 파악 및 해결**
- API 제공 업체 확인
- 서버 상태 확인
- 네트워크 상태 확인

**Step 4: 복구 완료 공지**
- "API 연결 복구 완료"

---

### 2. 대량 문의 발생

#### 대응 프로세스

**Step 1: 상황 파악**
- 같은 유형의 문의가 대량 발생
- 공통 이슈 확인

**Step 2: 통합 답변 준비**
- 대표 답변 작성
- 공지사항 등록

**Step 3: 개별 답변**
- 각 문의에 "공지사항 참고" 답변 + 링크
