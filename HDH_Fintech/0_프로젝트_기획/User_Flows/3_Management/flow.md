# User Flow #3: 관리 프로세스 (Management Flow)

> **최종 수정:** 2025-12-09
> **버전:** 1.0

---

## 1. 플로우 개요

**목적:** HDH 솔루션 운영자가 자신의 운영 성과를 관리하고 추적한다.

**시작:** Dashboard 우측 사이드바 관리 메뉴 접근
**종료:** 수익/손실 데이터 입력 및 조회 완료
**예상 소요 시간:** 5-10분 (데이터 입력/조회 시)

---

## 2. 정상 플로우 (Normal Flow)

### **[관리 메뉴 접근]**

#### Step 1: Dashboard에서 관리 메뉴 확인
- **위치:** Dashboard 우측 사이드바
- **표시 메뉴:**
  - 📊 My 누적 손익
  - 📝 My 손익 저널
  - ⚙️ My Settings

---

### **[관리 1] My 누적 손익**

#### Step 2-1: 누적 손익 페이지 진입
- **액션:** "My 누적 손익" 메뉴 클릭
- **페이지:** `/my-cumulative-pl`

#### Step 2-2: 누적 손익 조회
**표시 내용:**
- 전체 기간 누적 손익
- 월별 손익 요약
- 수익률 그래프
- 최근 거래 통계

**조회 옵션:**
- 기간별 필터링 (일별/주별/월별/전체)
- 차트 형식 변경 (선형/막대)
- 데이터 내보내기 (CSV/Excel)

#### Step 2-3: 누적 손익 데이터 입력/수정
**입력 항목:**
- 날짜
- 손익 금액
- 거래 건수
- 메모

**액션:**
- "새 데이터 추가" 버튼 클릭
- 정보 입력
- "저장" 버튼 클릭

#### Step 2-4: 저장 완료
- **시스템 응답:**
```
✅ 누적 손익 데이터가 저장되었습니다.
```

**결과:** 누적 손익 데이터가 업데이트되고 차트에 반영됨

---

### **[관리 2] My 손익 저널**

#### Step 3-1: 손익 저널 페이지 진입
- **액션:** "My 손익 저널" 메뉴 클릭
- **페이지:** `/my-pl-journal`

#### Step 3-2: 손익 저널 조회
**표시 내용:**
- 날짜별 거래 기록
- 손익 상세 내역
- 트레이딩 노트
- 학습 포인트

**정렬/필터 옵션:**
- 날짜순/손익순 정렬
- 수익/손실 필터링
- 기간별 검색

#### Step 3-3: 새 저널 작성
**액션:**
- "새 저널 작성" 버튼 클릭
- 저널 작성 폼 표시

**입력 항목:**
- 날짜
- 거래 내역
- 손익 금액
- 전략 분석
- 감정 상태
- 배운 점
- 개선할 점

#### Step 3-4: 저널 저장
- **액션:** "저장" 버튼 클릭
- **시스템 응답:**
```
✅ 손익 저널이 저장되었습니다.

💡 팁:
- 꾸준한 저널 작성은 트레이딩 실력 향상에 도움이 됩니다.
- 과거 저널을 복습하며 패턴을 발견해보세요.
```

**결과:** 저널이 저장되고 목록에 추가됨

#### Step 3-5: 기존 저널 수정/삭제
**수정:**
- 저널 항목 클릭
- 내용 수정
- "저장" 버튼 클릭

**삭제:**
- 저널 항목 선택
- "삭제" 버튼 클릭
- 확인 팝업에서 "확인" 클릭

---

### **[관리 3] My Settings**

#### Step 4-1: 설정 페이지 진입
- **액션:** "My Settings" 메뉴 클릭
- **페이지:** `/settings`

#### Step 4-2: 설정 항목 확인
**설정 카테고리:**

**1. 계좌 정보**
- 연결된 계좌 목록
- 계좌 추가/삭제
- 계좌 정보 수정

**2. API 연결**
- API 키 입력/수정
- 연결 상태 확인
- API 키 재발급

**3. 트레이딩 파라미터**
- 리스크 관리 설정
- 포지션 크기 설정
- 손절/익절 레벨

**4. 알림 설정**
- 이메일 알림 (수익/손실 알림)
- 시스템 알림
- 중요 공지 알림

**5. 프로필 정보**
- 닉네임 수정
- 비밀번호 변경
- 전화번호 변경

#### Step 4-3: 설정 수정
**액션:**
- 원하는 설정 항목 클릭
- 값 수정
- "저장" 버튼 클릭

**시스템 응답:**
```
✅ 설정이 저장되었습니다.

📌 참고:
- API 연결 변경 시 AI 프로그램을 재시작해주세요.
- 트레이딩 파라미터는 신중하게 변경해주세요.
```

#### Step 4-4: API 연결 테스트
**액션:**
- "연결 테스트" 버튼 클릭
- 시스템이 API 연결 확인

**연결 성공:**
```
✅ API 연결이 정상적으로 작동합니다.

연결 정보:
- 계좌: ********1234
- 상태: 활성
- 마지막 확인: 2025-12-09 14:30
```

**연결 실패:**
```
❌ API 연결에 실패했습니다.

원인:
- API 키가 유효하지 않거나
- 네트워크 연결 문제일 수 있습니다.

[매뉴얼 보기] [다시 시도]
```

---

### **[관리 4] 성과 추적 및 분석**

#### Step 5: 종합 대시보드 확인
- **위치:** Dashboard 메인 영역
- **표시 내용:**
  - 오늘의 손익 요약
  - 이번 주 손익 요약
  - 이번 달 손익 요약
  - 최근 저널 미리보기
  - 주요 지표 (승률, 평균 수익/손실 비율 등)

**결과:** 운영 성과를 한눈에 파악

---

## 3. 예외 플로우 (Exception Flows)

### 3-1. 데이터 입력 오류
- **조건:** 잘못된 형식의 데이터 입력
- **시스템 응답:**
```
❌ 입력 형식이 올바르지 않습니다.

- 날짜: YYYY-MM-DD 형식
- 금액: 숫자만 입력
```

### 3-2. 중복 데이터 입력
- **조건:** 같은 날짜에 중복 데이터 입력 시도
- **시스템 응답:**
```
⚠️ 해당 날짜에 이미 데이터가 있습니다.

[기존 데이터 수정] [새로 추가] [취소]
```

### 3-3. API 연결 끊김
- **조건:** 설정 페이지에서 API 연결 테스트 실패
- **시스템 응답:**
```
❌ API 연결에 실패했습니다.

확인 사항:
1. API 키가 정확한지 확인
2. 네트워크 연결 상태 확인
3. 계좌 상태 확인

[매뉴얼 보기] [Sunny에게 문의하기]
```

### 3-4. 데이터 삭제 확인
- **조건:** 데이터 삭제 시도
- **시스템 응답:**
```
⚠️ 정말 삭제하시겠습니까?

삭제된 데이터는 복구할 수 없습니다.

[취소] [삭제]
```

## 5. 성공 기준

### 5-1. 기능적 성공
- ✅ 사용자가 누적 손익을 입력하고 조회할 수 있다
- ✅ 사용자가 손익 저널을 작성하고 관리할 수 있다
- ✅ 사용자가 설정을 변경하고 저장할 수 있다
- ✅ API 연결 상태를 확인할 수 있다
- ✅ 데이터가 그래프와 차트로 시각화된다

### 5-2. UX 성공
- ✅ 데이터 입력 과정이 간단하고 직관적이다
- ✅ 그래프와 통계가 명확하게 표시된다
- ✅ 설정 변경이 쉽고 안전하다
- ✅ 오류 메시지가 명확하고 해결 방법을 제시한다

### 5-3. 비즈니스 성공
- ✅ 사용자가 자신의 운영 성과를 체계적으로 관리할 수 있다
- ✅ 데이터 기반 의사결정을 지원한다
- ✅ 지속적인 성과 개선을 유도한다
